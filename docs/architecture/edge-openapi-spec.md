# Edge OpenAPI Contract

## Purpose

Define the production-safe OpenAPI 3.1 and Swagger UI model for `awcms-edge/` without weakening the canonical AWCMS authority chain.

## Core Rules

- OpenAPI is descriptive only.
- Supabase remains the authority for Auth, Postgres, tenant isolation, RLS, and ABAC.
- Swagger UI must not expose internal-only maintenance or webhook routes in public runtime docs.
- Public and admin runtime docs must not expose service-role or infrastructure auth schemes.
- Internal edge docs are generated as artifacts only and are never mounted as runtime routes.

## Runtime Surfaces

| Surface | Route | Runtime Exposure | Notes |
| --- | --- | --- | --- |
| Health | `/health` | Public | Worker liveness |
| Public OpenAPI | `/openapi/public.json` | Public | Public boundary only |
| Public Swagger UI | `/docs` | Public | Loads the public spec only |
| Admin OpenAPI | `/openapi/admin.json` | Protected | Requires bearer auth, platform scope, and `platform.docs.read` |
| Admin Swagger UI | `/docs/admin` | Protected | Loads the admin spec only |
| Internal OpenAPI artifact | `awcms-edge/openapi/internal.json` | Not exposed | Generated by `npm run openapi:build` |

## Public Route Contract Notes

- Public tenant-aware routes may resolve tenant identity from `tenantId`, `tenant_id`, or `domain`, depending on the route contract.
- When both a tenant identifier and a domain are supplied, the runtime contract requires both values to resolve to the same tenant; otherwise the documented failure mode is `400 Tenant/domain mismatch`.
- Public media delivery is intentionally narrower than generic object lookup: `/public/media/{storageKey}` only documents canonical public keys under `tenants/<tenant_id>/...`.
- Keys targeting `tenants/<tenant_id>/protected/...`, malformed segments, or traversal-like values are documented as invalid request paths rather than “not found” cases.
- Compatibility routes such as `/functions/v1/extensions/events/public`, `/functions/v1/extensions/public-modules`, and `/functions/v1/serve-sitemap` must describe the same tenant/domain guardrails as their maintained public counterparts.

## Trust Boundaries

| Boundary | Intended Consumers | Examples | Runtime Exposure |
| --- | --- | --- | --- |
| `public` | Browsers, public portals, anonymous callers | `/public/sitemap`, Turnstile verify, public extension feeds | Public |
| `admin` | Authenticated operators | media upload orchestration, public rebuild, extension lifecycle | Protected |
| `internal` | Local maintenance, server-to-server callbacks, operational tooling | local media reconciliation, secret rebuild webhooks | Artifact only |

Boundary assignment is based on operational trust, not path naming alone.

## Security Schemes

| Scheme | Specs | Purpose |
| --- | --- | --- |
| `supabaseBearerAuth` | admin, internal | Authenticated operator access through Supabase bearer tokens |
| `secretHeaderAuth` | internal only | Shared-secret callback and webhook flows |

Disallowed in public or admin docs:

- service-role secrets
- infrastructure tokens
- secret-bound internal auth schemes

## Vendor Extensions

Each documented operation may declare the following AWCMS metadata:

| Extension | Meaning |
| --- | --- |
| `x-awcms-boundary` | `public`, `admin`, or `internal` |
| `x-awcms-scope` | `public`, `tenant`, or `platform` |
| `x-awcms-permission` | Required ABAC permission string for protected mutations |
| `x-awcms-tenant-context` | `required`, `optional`, or `none` |
| `x-awcms-r2` | Whether the route interacts with Cloudflare R2 |
| `x-awcms-soft-delete-aware` | Whether the route depends on soft-delete filtering |

## Validation Commands

```bash
cd awcms-edge
npm run openapi:build
npm run openapi:validate
npm run openapi:diff
npm run test
npm run typecheck
```

Expected outcomes:

- public, admin, and internal specs build successfully
- public and admin specs contain no internal routes
- admin docs routes reject anonymous access
- no internal docs runtime routes exist
- no service-role or secret-header schemes appear in public or admin docs

## Operational Notes

- `/docs` disables interactive submission by default.
- `/docs/admin` is protected before Swagger UI is rendered.
- `awcms-edge/openapi/internal.json` may be used by CI, reviewers, or internal tooling, but it is not a runtime API surface.
- Any new documented admin mutation must carry accurate permission metadata and preserve the Supabase enforcement model.
- Route descriptions should explicitly call out public guardrails when runtime behavior depends on canonical storage-key shape, tenant/domain resolution, or mismatch rejection.
